# Развертывание на Vercel

Ваш проект готов к развертыванию на Vercel!

## Предварительные требования
1.  Аккаунт на [Vercel](https://vercel.com).
2.  Ваш **API-ключ Groq**.

## Вариант А: Развертывание через Vercel CLI (Рекомендуется)

1.  **Установите Vercel CLI**:
    ```bash
    npm install -g vercel
    ```

2.  **Войдите в систему**:
    ```bash
    vercel login
    ```

3.  **Разверните проект**:
    Запустите эту команду в папке вашего проекта:
    ```bash
    vercel
    ```

4.  **Настройка**:
    -   Vercel задаст несколько вопросов. Нажимайте `Enter`, чтобы принять значения по умолчанию.
    -   **Важно**: Когда спросят, **свяжите (link) ваш проект**.

5.  **Установка переменных окружения**:
    После первой попытки развертывания (или во время настройки), перейдите в панель управления Vercel:
    -   Выберите ваш проект.
    -   Перейдите в **Settings** (Настройки) > **Environment Variables** (Переменные окружения).
    -   Добавьте `GROQ_API_KEY` со значением вашего API-ключа.
    -   Переразверните проект при необходимости.

## Вариант Б: Развертывание через GitHub

1.  Загрузите ваш код в репозиторий GitHub.
2.  Перейдите в [Панель управления Vercel](https://vercel.com/dashboard) и нажмите **"Add New..."** > **"Project"**.
3.  Импортируйте ваш репозиторий GitHub.
4.  В разделе "Environment Variables" (Переменные окружения) добавьте `GROQ_API_KEY`.
5.  Нажмите **Deploy** (Развернуть).

## ⚠️ Важное примечание о данных
Vercel — это "serverless" (бессерверная) платформа. Файлы, которые вы сохраняете (например, календарь), являются **временными**.
-   **Поведение**: Ваш график будет сохраняться в `/tmp`.
-   **Сохранность**: Данные будут удалены, когда сервер "уснет" (обычно после нескольких минут бездействия).
-   **Решение**: Для реального приложения вам понадобится база данных (например, MongoDB или PostgreSQL). Для этой демонстрации использование `/tmp` работает, но данные не сохраняются надолго.
